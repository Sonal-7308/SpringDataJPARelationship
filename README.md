SpringDataJPA : Unidirectional OneToMany Relationship

A simple console based application using SpringDataJPA. Implement a unidirectional One-to-Many relationship from Department to Employee.

Each department can have multiple employees, but an employee belongs to only one department.

Your app should have the following CRUD functionality:

Create a new department with multiple employees.

Retrieve all departments along with their employees.

Update the name of a department.

Add a new employee to an existing department.

Remove an employee from a department.

Delete a department along with all its employees.
